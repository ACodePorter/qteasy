# qteasy 文档国际化（i18n）工作流

本文说明 qteasy Sphinx 文档的多语言构建与维护流程。  
术语约束见 [`I18N_GLOSSARY.md`](I18N_GLOSSARY.md)。

---

## 1. 目录与语言

| 项 | 路径 / 值 |
|----|-----------|
| 源文档（基准语言） | [`docs/source/`](source/)，`language = 'zh_CN'` |
| 翻译目录 | [`docs/source/locale/<lang>/LC_MESSAGES/`](source/locale/) |
| Sphinx 配置 | [`docs/source/conf.py`](source/conf.py) |
| 当前语言包 | `en`、`de`、`fr`、`es`、`zh_TW`、`ja` |
| 西班牙语 | ISO 代码为 **es**（非 sp） |

Read the Docs 已配置多语言翻译项目；各语言独立 URL，例如：

- 简体：`https://qteasy.readthedocs.io/zh-cn/latest/`
- 英文：`https://qteasy.readthedocs.io/en/latest/`

## 3. 常用 Make 目标

在 `docs/` 目录下执行：

| 命令 | 作用 |
|------|------|
| `make gettext` | 从 zh_CN 源提取 `.pot` 到 `build/gettext/` |
| `make update-po` | 将 `.pot` 合并到 `en de fr es` 的 `.po`（保留已有译文） |
| `make i18n-stats` | 打印各语言翻译完成度基线 |
| `make html-en` | 构建英文 HTML 预览（`build/html/en/`） |
| `make html-de` / `html-fr` / `html-es` | 其它语言 HTML 预览 |
| `make html-zh-tw` / `html-ja` | 繁中 / 日语 HTML 预览 |

指定语言更新 `.po`：

```bash
cd docs
make gettext
I18N_LANGS="en de" make update-po
```

Phase 3 新增语言：

```bash
sphinx-intl update -p build/gettext -l zh_TW -l ja
```

---

## 4. 源文档变更后的维护流程

1. 修改 [`docs/source/`](source/) 下的 `.md` / `.rst`（zh_CN）。
2. 运行 `make gettext && make update-po`。
3. **至少**更新对应条目的 **en** 译文；其它语言可暂留 untranslated。
4. 用 Poedit 或编辑器翻译/校对 `.po`，清除 `#, fuzzy`。
5. 运行 `make i18n-stats` 确认进度；可选 `make html-en` 本地预览。
6. 重大发布前对比 `i18n-stats` 输出，确保 en 无回退。

## 7. 编辑器建议

历史 `.po` 多由 **Poedit 3.9** 维护。推荐：

- 用 Poedit 打开单个 `.po` 或整个 `LC_MESSAGES` 目录；
- 启用翻译记忆，复用 `manage_data` 重复字段描述；
- 翻译时对照 [`I18N_GLOSSARY.md`](I18N_GLOSSARY.md) 中的术语与内链规则。
